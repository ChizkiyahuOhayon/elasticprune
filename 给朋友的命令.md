# Round 3：GQA 4000 oracle + attention signals

目标：跑新版 oracle，样本数扩大到 4000，并保存 selector attention concentration signals：

- `attn_entropy_norm`
- `attn_effective_support_norm`
- `attn_gini`
- `attn_top1_mass`
- `attn_top5_mass`
- `attn_top10_mass`
- `attn_top25_mass`

这些字段用于后续判断新 router signal 是否有效。

## 1. 更新代码

```bash
cd elasticprune
git pull
git log --oneline -1
```

确认最新提交至少是：

```bash
9d40dda Add attention concentration signals for router search
```

如果看到更新的提交也没问题。

## 2. 激活环境

```bash
conda activate elastic
export HF_ENDPOINT=https://hf-mirror.com
```

## 3. 一条命令运行 Round 3

```bash
bash scripts/run_gqa4000_attn_oracle.sh
```

脚本会自动：

1. 创建 `results_gqa4000_attn/`
2. 启动 8 个 shard
3. 把日志保存到 `results_gqa4000_attn/logs_oracle_*.log`
4. 等所有 shard 结束
5. 检查 `oracle.shard*.json` 是否齐全
6. 跑 oracle 分析
7. 跑 offline router analysis
8. 跑 train/test router search
9. 打包 `oracle_results_gqa4000_attn.tar.gz`

## 4. 监控

另开一个终端：

```bash
tail -f results_gqa4000_attn/logs_oracle_0.log
nvidia-smi
```

## 5. 跑完后发回

把这个文件发回来：

```bash
oracle_results_gqa4000_attn.tar.gz
```

## 6. 如果中途失败

先看哪个 shard 没有生成：

```bash
ls -lh results_gqa4000_attn/oracle.shard*.json
tail -80 results_gqa4000_attn/logs_oracle_0.log
```

如果只是某个 shard 失败，例如 shard 3，可以单独补跑：

```bash
CUDA_VISIBLE_DEVICES=3 nohup python scripts/oracle_gqa.py \
  --n 4000 --shard 3 --num-shards 8 \
  --out results_gqa4000_attn/oracle.json \
  > results_gqa4000_attn/logs_oracle_3.rerun.log 2>&1 &
```

全部 shard 齐后，再手动分析和打包：

```bash
python scripts/analyze_oracle.py --dir results_gqa4000_attn

python scripts/analyze_router_offline.py \
  --dir results_gqa4000_attn \
  --out results_gqa4000_attn/router_offline.json \
  --csv results_gqa4000_attn/router_labeled_samples.csv \
  --md results_gqa4000_attn/router_offline.md

python scripts/search_router_offline.py \
  --dir results_gqa4000_attn \
  --out results_gqa4000_attn/router_search.json \
  --md results_gqa4000_attn/router_search.md

tar czf oracle_results_gqa4000_attn.tar.gz results_gqa4000_attn/
```
