# 直接复制粘贴到终端（按顺序）

前提：机器有 conda 和 CUDA 驱动。所有命令针对国内网络环境已配好镜像。

## 1. 克隆 + 装环境（约 20-40 分钟，多数时间在下载）

```bash
git clone https://github.com/ChizkiyahuOhayon/elasticprune.git
cd elasticprune
bash setup_env.sh cn
```

如果 GitHub 克隆超时，换：
```bash
git clone https://ghproxy.net/https://github.com/ChizkiyahuOhayon/elasticprune.git
```

## 2. 冒烟测试（约 5 分钟，含首次下载 llava-1.5-7b 约 14GB）

```bash
conda activate elastic
export HF_ENDPOINT=https://hf-mirror.com
CUDA_VISIBLE_DEVICES=0 python -m elasticprune.smoke_test
```

**期望输出**：先打印一段完整回答（关于图里的猫），然后 keep 0.5/0.1/0.05 三档剪枝的回答，最后一行 `冒烟测试通过 ✔`。
报错的话把完整 traceback 截图发回来。

## 3. 8 卡跑 oracle 实验（约 2-4 小时）

```bash
mkdir -p results
for i in 0 1 2 3 4 5 6 7; do
  CUDA_VISIBLE_DEVICES=$i nohup python scripts/oracle_gqa.py \
    --n 2000 --shard $i --num-shards 8 --out results/oracle.json \
    > logs_oracle_$i.log 2>&1 &
done
```

监控进度：
```bash
tail -f logs_oracle_0.log        # 看单卡进度
nvidia-smi                        # 确认 8 卡都在跑
```

## 4. 跑完后把结果发回来

```bash
tar czf oracle_results.tar.gz results/ logs_oracle_*.log
```

把 `oracle_results.tar.gz`（很小，几 MB）通过微信/网盘发回来即可。

## 常见问题

- **模型下载卡住**：确认 `echo $HF_ENDPOINT` 输出 `https://hf-mirror.com`，不是就重新 export
- **CUDA OOM**：不应该发生（7B bf16 约 15GB / 24GB），若发生检查该卡是否有别的进程 `nvidia-smi`
- **flash-attn 编译失败**：忽略，本阶段用不到
- **某个 shard 挂了**：单独重跑那一个 `CUDA_VISIBLE_DEVICES=X python scripts/oracle_gqa.py --n 2000 --shard X --num-shards 8 --out results/oracle.json`
