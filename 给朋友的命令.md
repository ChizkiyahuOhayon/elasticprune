# 第二轮实验命令：GQA budget-response v2

目标：重新跑增强版 oracle。新版会保存每个预算的 prediction text、true full-generate 结果、preservation/correction/non-monotonic 分析所需字段。

## 0. 更新代码

如果已经 clone 过仓库：

```bash
cd elasticprune
git pull
```

如果还没 clone：

```bash
git clone https://github.com/ChizkiyahuOhayon/elasticprune.git
cd elasticprune
bash setup_env.sh cn
```

如果 GitHub 克隆超时，换：

```bash
git clone https://ghproxy.net/https://github.com/ChizkiyahuOhayon/elasticprune.git
cd elasticprune
bash setup_env.sh cn
```

## 1. 激活环境

```bash
conda activate elastic
export HF_ENDPOINT=https://hf-mirror.com
```

## 2. 清理旧结果

只删本项目实验输出，不影响代码：

```bash
rm -rf results logs_oracle_*.log oracle_results_v2.tar.gz
mkdir -p results
```

## 3. 8 卡跑新版 oracle

这轮比第一轮多保存 prediction 和 full-generate 检查，会稍慢一些。预计 3-6 小时，具体看机器 IO 和模型缓存。

```bash
for i in 0 1 2 3 4 5 6 7; do
  CUDA_VISIBLE_DEVICES=$i nohup python scripts/oracle_gqa.py \
    --n 2000 --shard $i --num-shards 8 --out results/oracle.json \
    > logs_oracle_$i.log 2>&1 &
done
```

监控：

```bash
tail -f logs_oracle_0.log
nvidia-smi
```

如果某张卡挂了，只重跑对应 shard，例如 shard 3：

```bash
CUDA_VISIBLE_DEVICES=3 python scripts/oracle_gqa.py \
  --n 2000 --shard 3 --num-shards 8 --out results/oracle.json
```

## 4. 本机先做一次分析

全部跑完后执行：

```bash
python scripts/analyze_oracle.py --dir results
```

正常会看到：

- fixed ratio accuracy
- true full_generate accuracy
- task oracle
- preservation oracle
- correction oracle samples
- non-monotonic samples
- `results/oracle_analysis_v2.png`

## 5. 打包发回

```bash
tar czf oracle_results_v2.tar.gz results/ logs_oracle_*.log
```

把 `oracle_results_v2.tar.gz` 发回来即可。

## 6. 如果显存或速度有问题

先跑不计算额外信号的版本：

```bash
rm -rf results logs_oracle_*.log
mkdir -p results
for i in 0 1 2 3 4 5 6 7; do
  CUDA_VISIBLE_DEVICES=$i nohup python scripts/oracle_gqa.py \
    --n 2000 --shard $i --num-shards 8 --out results/oracle.json --skip-signals \
    > logs_oracle_$i.log 2>&1 &
done
```

这个版本仍然会保存 prediction/full/oracle 关键字段，只是没有 redundancy/query-specificity 信号。
